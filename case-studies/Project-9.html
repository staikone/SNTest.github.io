Rebooking Option for Past Trips

Adding a “Check Availability” CTA to Airbnb’s past reservation UI

Overview

Many users revisit places they enjoyed and want to rebook them. On Airbnb’s “Past Reservation” view, there is no visible path to check availability or rebook directly. My task was to design a minimal but effective enhancement — a CTA integrated into the existing reservation view — that encourages rebooking while respecting Airbnb’s UI style and minimizing visual noise.

Problem

When viewing a past reservation, users see useful details (dates, host, instructions), but there is **no way** to check availability or rebook. This creates friction for loyal users wanting to revisit favorite stays. The absence of this functionality undermines retention and user loyalty.

Process

  1. UX Audit & Needs Analysis: Analyzed the reservation-details screen, identified user pain points, and noted that frequent users expect rebooking options.
  2. Design Constraints & Style Matching: Reviewed Airbnb’s UI language, tone, and component standards to ensure consistency.
  3. CTA Positioning Ideation: Sketched multiple placements for the “Check availability” button near the property header, above the trip summary, or within the reservation card.
  4. Visual Design & Interaction: Created button variants, hover/tap states, and responsive adjustments. Ensured legibility and contrast in both mobile and desktop layouts.
  5. Prototyping & Review: Built mockups for both views, reviewed with stakeholders for clarity and minimal disruption, and refined positioning and spacing.

Solution

The solution introduces a new CTA section labeled “Loved this place?” right above the trip summary. Users see a clear “Check Availability” button that leads to a rebooking flow. On desktop, hovering reveals subtle feedback; on mobile/tablet, tapping opens a modal. The design preserves Airbnb’s visual language and integrates seamlessly without disrupting existing layout.

Results

  • 🚀 Rebooking path becomes one click away, reducing friction in user journey.
  • 🎯 Maintains Airbnb’s clean and lightweight UI without adding clutter.
  • 💡 Encourages loyalty-driven behavior by enabling repeat bookings.
  • 📐 Compatible with responsive views: desktop, tablet, mobile.

Learnings

Even large platforms can overlook high-retention touchpoints. Working within brand constraints taught me how to add value with micro-enhancements. Under an NDA, I couldn’t share performance metrics; moving forward, I’ll push for A/B testing, click-through data, and integration into analytics from pilot releases to strengthen the UX narrative.
